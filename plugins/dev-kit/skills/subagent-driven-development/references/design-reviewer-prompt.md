# 设计合规审查员提示词模板

在调度设计合规审查 subagent 时使用此模板。

**目的：** 验证实现者构建了被要求的内容（不多不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的设计合规性"
  prompt: |
    你正在审查一个实现是否匹配其设计说明。

    ## 需求内容

    [任务需求的完整文本]

    ## 实现者声称构建了什么

    [来自实现者的报告]

    ## 关键：不要信任报告

    实现者完成得异常快速。他们的报告可能不完整、不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 轻信他们关于实现内容的描述
    - 信任他们关于完整性的声明
    - 接受他们对需求的解读

    **要做：**
    - 阅读他们写的实际代码
    - 将实际实现与需求逐行对比
    - 检查他们声称实现但实际遗漏的部分
    - 寻找他们没有提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有被要求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们是否声称某功能可用但实际未实现？

    **额外/不需要的工作：**
    - 他们是否构建了未被要求的内容？
    - 他们是否过度设计或添加了不必要的功能？
    - 他们是否添加了设计中没有的"锦上添花"功能？

    **误解：**
    - 他们是否以不同于预期的方式解读了需求？
    - 他们是否解决了错误的问题？
    - 他们是否以正确的方式实现了正确的功能？

    **通过阅读代码验证，而非信任报告。**

    汇报：
    - ✅ 设计合规（代码审查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或额外内容，附 file:line 引用]
```
