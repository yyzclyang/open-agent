# 快速原型

当目标是快速验证产品想法、界面方案或交互流程时，使用此技能输出高保真 HTML 原型。

## 适用场景

- MVP 开发
- 概念验证
- 技术探索
- 快速演示原型
- 用户需求验证
- 技术选型验证

## 工作原则

- 快速迭代：优先验证核心路径
- 最小可行：只做必要内容，避免过度设计
- 技术简化：优先使用成熟、低摩擦方案
- 重交互验证：原型应可点击、可演示，而不是纯静态稿
- 只输出 HTML 原型，不扩展到 React、小程序或其他正式技术栈

## 工作流程

1. 先理解产品目标、主要用户和核心业务流程。
2. 提炼 3-4 个最高优先级页面，避免一开始铺太大。
3. 设计接近真实产品的高保真界面，优先保证信息架构和交互路径。
4. 使用 HTML + Tailwind CSS 生成页面；需要时可使用 Font Awesome。
5. 每个页面使用独立 HTML 文件，包含自身所需的样式和脚本。
6. 加入基础交互和数据逻辑，使原型具备演示价值。

## 输出约束

- 如无特别要求，最多输出 4 个页面
- 移动端按 iPhone 15 Pro 尺寸考虑，桌面端按 1440px 宽度适配
- 图片应使用真实资源，不使用无意义占位图
- 优先保证真实感、完整性和可演示性

## 交付内容

- 用户体验分析
- 界面与页面规划
- 高保真 HTML 原型文件
- 核心交互说明
