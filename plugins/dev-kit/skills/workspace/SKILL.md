# 工作区管理

当需要把开发任务隔离到独立工作区时，使用此技能管理基于 `git worktree` 的工作区生命周期。

## 创建工作区

目标：创建一个独立 worktree，用于隔离某项开发任务。

### 执行步骤

1. 检测当前目录是否为 git 仓库。
2. 确认当前工作目录干净，没有未提交改动。
3. 识别主分支，通常为 `main` 或 `master`。
4. 更新主分支，除非用户明确要求跳过。
5. 生成或使用用户提供的工作区名称。
6. 创建 worktree 到 `.y-commands-workspaces/{name}`。
7. 在 `.y-commands-workspaces/.metadata` 中记录原始分支、创建时间等元数据。
8. 将 `/.y-commands-workspaces` 加入 `.git/info/exclude`。

## 完成工作区

目标：结束一个已完成的工作区，并选择合适的集成方式。

### 执行步骤

1. 检测当前是否位于仓库根目录，而不是 workspace 内。
2. 列出所有可用工作区。
3. 如果有多个工作区，让用户选择目标。
4. 读取元数据，确定原始分支。
5. 提供后续处理方式：
   - 合并回原始分支
   - 创建 GitHub PR
6. 按用户选择执行对应流程。

## 使用要求

- 在执行任何 worktree 相关操作前，先确认不会覆盖用户未提交的工作
- 如果元数据缺失、分支状态异常或工作区不干净，应先停止并提示风险
- 优先保证隔离性和可回退性，避免在主工作目录直接操作
