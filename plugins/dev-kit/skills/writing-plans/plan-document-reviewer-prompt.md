# 计划文档审阅者提示模板

在派发计划文档审阅者子代理时使用此模板。

**用途：** 验证计划是否完整、与规格匹配，且任务分解合理。

**派发时机：** 完整计划编写完成之后。

```
Task tool (general-purpose):
  description: "审阅计划文档"
  prompt: |
    你是一个计划文档审阅者。验证此计划是否完整且已准备好进入实施阶段。

    **待审阅计划：** [PLAN_FILE_PATH]
    **参考设计：** [DESIGN_FILE_PATH]

    ## 检查内容

    | 类别 | 要查找的内容 |
    |------|-------------|
    | 完整性 | TODO、占位符、不完整的任务、缺失的步骤 |
    | 规格对齐 | 计划覆盖了规格需求，无重大范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤可操作 |
    | 可构建性 | 工程师能否按此计划执行而不卡住？ |

    ## 校准原则

    **只标记会在实施阶段造成真正问题的缺陷。**
    实施者构建了错误的东西或卡住了——这些是问题。
    措辞上的小改进、风格偏好、"锦上添花"的建议——这些不是问题。

    除非有严重缺陷——规格需求遗漏、矛盾的步骤、占位符内容、
    或任务太模糊无法执行——否则应该批准。

    ## 输出格式

    ## 计划审阅

    **状态：** 已批准 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [对实施的影响]

    **建议（仅供参考，不阻塞批准）：**
    - [改进建议]
```

**审阅者返回：** 状态、问题（如有）、建议
