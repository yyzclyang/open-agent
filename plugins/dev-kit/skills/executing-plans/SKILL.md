# 执行计划

## 概述

加载计划，批判性审阅，执行所有任务，完成后报告。

**开始时宣布：** "我正在使用 executing-plans 技能来实施此计划。"

**注意：** 告诉你的人类伙伴，在有子代理支持的情况下效果更好。如果在支持子代理的平台（如 Claude Code 或 Codex）上运行，工作质量会显著提高。如果子代理可用，请使用 dev-kit:subagent-driven-development 而非此技能。

## 流程

### 步骤 1：加载并审阅计划
1. 读取计划文件
2. 批判性审阅——识别关于计划的任何问题或疑虑
3. 如果有疑虑：在开始前向人类伙伴提出
4. 如果没有疑虑：创建 TodoWrite 并继续

### 步骤 2：执行任务

对每个任务：
1. 标记为进行中
2. 严格按每个步骤执行（计划包含小口吞咽的步骤）
3. 按要求运行验证
4. 标记为已完成

### 步骤 3：完成开发

所有任务完成并验证后：
- 宣布："我正在完成了此工作。"

## 何时停下来求助

**以下情况立即停止执行：**
- 遇到阻碍（缺少依赖、测试失败、指令不清晰）
- 计划存在关键缺陷导致无法开始
- 不理解某个指令
- 验证反复失败

**宁可询问澄清，也不要猜测。**

## 何时回到早期步骤

**回到审阅（步骤 1）的情况：**
- 伙伴根据你的反馈更新了计划
- 根本方案需要重新思考

**不要硬闯阻碍**——停下来问。

## 记住
- 先批判性审阅计划
- 严格按照计划步骤执行
- 不要跳过验证
- 计划要求时引用相关技能
- 遇到阻碍就停下，不要猜测
- 未经用户明确同意，绝不在 main/master 分支上开始实施

## 集成

**必需的工作流技能：**
- **dev-kit:writing-plans** - 创建此技能执行的计划
