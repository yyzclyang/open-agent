# 保存设计文档

将 brainstorming 会话保存为设计文档。

**用法：** `/design`

**步骤：**
1. 定位当前会话历史中的 `/brainstorm` 调用
2. 收集该命令之后的所有用户和助手消息
3. 分析对话以推断正在设计的主要功能/主题
4. 将 brainstorming 内容转换为结构化的设计文档
5. 从主题生成简短描述性 slug（3-5 个词，小写，连字符分隔）
6. 获取当前日期（YYYY-MM-DD 格式）
7. 构建文件名：`docs/designs/YYYY-MM-DD-<slug>.md`
8. 如果同名文件已存在，追加后缀如 -1, -2 或短时间戳
9. 如果 `docs/designs` 目录不存在，创建它
10. 使用 write 工具创建文件并写入设计内容
11. 最后，打印最终写入的文件路径

如果历史中未找到 `/brainstorm` 命令，明确告知用户未检测到 brainstorming 会话，不写入任何文件。

## 设计文档结构

设计文档必须遵循以下结构：

```markdown
# [功能名称]

**日期：** YYYY-MM-DD

## 上下文
基于早期 brainstorming 消息总结初始想法和动机。

## 讨论
总结对话中的关键问题、答案、权衡和探索的替代方案。

## 方案
总结最终达成的一致方向和如何解决问题。

## 架构
如果讨论了，描述技术细节、组件、流程和重要的实施注意事项。
```

## 记住
- 仅基于 `/brainstorm` 之后的内容
- 不要包含原始聊天日志；只包含提炼的设计
- 使用中文回复
