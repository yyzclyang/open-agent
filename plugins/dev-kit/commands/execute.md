# 执行实施计划

加载计划，批判性审查，批量执行任务，批次之间报告供审查。

**用法：** `/execute [plan-file]`

**核心原则：** 批量执行，在架构审查的检查点。

**开始时宣布：** "我正在实施这个计划。"

## 流程

### 步骤 1: 加载和审查计划
1. 读取计划文件（如果未指定，查找 `docs/plans/` 中最新的计划文件）
2. 批判性审查 - 识别计划的任何问题或担忧
3. 如果有担忧：在开始前向用户提出
4. 如果没有担忧：创建 TodoWrite 并继续

### 步骤 2: 执行批次
**默认：前 3 个任务**

对每个任务：
1. 标记为 in_progress
2. 严格按照每个步骤执行（计划有细粒度步骤）
3. 运行指定的验证
4. 标记为 completed

### 步骤 3: 报告
批次完成时：
- 显示实现的内容
- 显示验证输出
- 说："准备接受反馈。"

### 步骤 4: 继续
基于反馈：
- 如果需要应用更改
- 执行下一批次
- 重复直到完成

### 步骤 5: 完成开发

所有任务完成并验证后：
- 宣布："我已完成此实施。"

## 何时停止并寻求帮助

**立即停止执行当：**
- 批次中遇到阻塞（缺少依赖、测试失败、指令不清楚）
- 计划有阻止开始的严重缺陷
- 不理解某个指令
- 验证反复失败

**询问澄清而不是猜测。**

## 何时回到早期步骤

**返回早期步骤当：**
- 批次中遇到阻塞 → 停止并报告
- 计划有缺陷 → 返回步骤 1 审查
- 验证失败 → 停止并询问

**参数：** $1
